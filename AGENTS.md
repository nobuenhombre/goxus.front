# goxus.front — AGENTS.md

## 1. Overview

This is the frontend for **goxus**, a web application. It lives as a git submodule at `github.com/nobuenhombre/goxus.front` inside the main `goxus` monorepo (`github.com/nobuenhombre/goxus`).

Stack: Next.js 15+ (App Router), TypeScript, Tailwind CSS v4, shadcn/ui (planned), React Compiler.

## 2. Tech Stack

| Tier          | Technology                                                |
|---------------|-----------------------------------------------------------|
| Framework     | [Next.js](https://nextjs.org/docs) (App Router)           |
| Language      | TypeScript                                                |
| Styling       | [Tailwind CSS v4](https://tailwindcss.com/docs)           |
| UI Library    | [shadcn/ui](https://ui.shadcn.com/docs) (planned/partial) |
| Compiler      | React Compiler (enabled in next.config)                   |
| Runtime       | Node.js v24.16.0 (managed via nvm)                        |

## 3. Architecture

```
src/
├── app/          # Next.js App Router — file-based routing
│   ├── layout.tsx    # Root layout (shared shell)
│   ├── page.tsx      # Home page
│   └── ...           # Additional route groups / pages
├── components/   # shadcn/ui components (registry)
│   └── ui/           # Generated by `npx shadcn add ...`
├── lib/          # Utility functions, constants, helpers
│   └── utils.ts      # cn() helper and shared utilities
└── styles/       # Global CSS / Tailwind entry point
    └── globals.css
```

- `src/app/` — every folder/file maps to a URL route (App Router convention).
- `src/components/ui/` — shadcn/ui components (button, card, dialog, etc.). Hand-authored app-specific components live in `src/components/` outside `ui/`.
- `src/lib/` — pure helpers, type definitions, API client wrappers.

## 4. Conventions

- **Naming**: PascalCase for components (`LoginForm.tsx`), kebab-case for utility files (`api-client.ts`), camelCase for functions/variables.
- **Imports**: Use absolute imports from `@/` (mapped to `src/`). Group: React → Next.js → third-party → local.
- **Component style**: Server Components by default; `"use client"` only when interactivity (hooks, event handlers, browser APIs) is required.
- **CSS**: Tailwind utility classes in JSX. Avoid custom CSS files unless absolutely necessary.
- **Types**: Define shared types in `src/lib/types.ts` or co-locate with the component.

## 6. Gotchas

- **shadcn/ui init**: Run `npx shadcn@latest init` in the front directory. It will create/overwrite `components.json` and prompt for style/base color. After init, add individual components with `npx shadcn@latest add button`.
- **Tailwind v4 vs v3**: Tailwind v4 changes the config format — no `tailwind.config.js` by default. Configuration uses CSS-based `@theme` in `globals.css`. The `@apply` directive still works, but many v3 configuration APIs (`theme.extend`, `plugins`) are replaced by CSS-first equivalents. If a component's docs reference `tailwind.config.js`, adapt the config to v4's `@theme` syntax.
- **nvm required**: Node.js v24.16.0 is project-specific. If you see engine errors, run `nvm use` before any npm command.
- **React Compiler**: Enabled in `next.config.ts` via the `experimental.reactCompiler` option. Avoid patterns that violate Rules of Hooks (no conditional `useState`/`useEffect`).
- **shadcn-admin reference**: [satnaing/shadcn-admin](https://github.com/satnaing/shadcn-admin) is used as a UI pattern reference but is not a dependency.
